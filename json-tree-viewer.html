JSON Flow Viewer Manual

Load a JSON file on the right to inspect it as a tree or as an Ingredient Scrap production flow.

Material Flow JSON

The tree mode can display any JSON. The production flow mode expects a root object with a flows array. Each flow represents one recipe step with ingredients, recipe metadata, and results.

Icon lookup order

The production flow checks every ingredient and result for an icon in this order:

entry.icon
entry.prototype.icon
entry.result.prototype.icon

A real image is rendered only after the icon has a url field. The URL can be present in the JSON already, or the viewer can create it from loaded ZIPs, dropped image files, asset_roots, or the Factorio root path.

Flow shape

  • flows: array of recipe flow entries.
  • recipe.name: recipe id shown in the center column.
  • ingredients and results: arrays shown left and right.
  • name, type, and amount are enough for a basic chip.
  • icon and prototype.icon are optional, but needed for visible icons.

Direct URL icon

{
  "name": "steam",
  "prototype": {
    "icon": {
      "url": "data:image/png;base64,..."
    }
  }
}

Entry-level icon

{
  "name": "steam",
  "icon": {
    "url": "icons/steam.png"
  }
}

Minimal example

{
  "schema": "ingredient-scrap-material-flow/v1",
  "flows": [
    {
      "material": "prefix-example",
      "recipe": {
        "name": "test-prefix-example-mixed",
        "main_product": "prefix-example-product-mixed"
      },
      "ingredients": [
        { "type": "fluid", "name": "molten-prefix-example", "amount": 20 },
        { "type": "item", "name": "prefix-example-plate", "amount": 5 }
      ],
      "results": [
        { "type": "item", "name": "prefix-example-product-mixed", "amount": 1 },
        { "type": "item", "name": "prefix-example-scrap", "amount_min": 1, "amount_max": 3 }
      ]
    }
  ]
}

Material-flow ingredient

{
  "type": "item",
  "name": "prefix-example-plate",
  "amount": 5,
  "prototype": {
    "type": "item",
    "name": "prefix-example-plate",
    "icon": {
      "path": "__example-mod__/graphics/icons/prefix-example-plate.png",
      "source": {
        "mod": "example-mod",
        "inner_path": "graphics/icons/prefix-example-plate.png"
      },
      "url": "data:image/png;base64,..."
    }
  }
}

path and source are metadata. They become visible only when the icon also has url, or when the viewer can resolve that metadata to a local file/blob URL.

Offline and ZIP workflow

  • JSON contains source.mod and source.inner_path.
  • The viewer can load a local mod ZIP and find the matching icon inside it.
  • The viewer then creates a blob: URL internally and writes it to icon.url.
  • Ingredient Scrap's test tool can also extract ZIP icons and provide asset_roots.

Useful additions

  • Relative URLs such as "url": "icons/prefix-example-plate.png" work when reachable from the HTML file.
  • Mipmap strips and non-square icons are displayed as square chips cropped from the left side.
  • If an icon is missing, hover the placeholder to see the unresolved path.
